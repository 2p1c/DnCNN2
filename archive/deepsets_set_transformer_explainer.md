# DeepSets 与 Set Transformer 为什么更适合当前任务

## 背景

当前项目的输入是单接收点时间序列，主网络是 1D 卷积自编码器：

- 输入形状是 `(B, 1, T)`
- 输出形状是 `(B, 1, T)`
- 网络默认只有一条时间轴，不包含显式空间轴

当后续扩展到多接收点时，数据的本质会从“固定长度时间序列”转成“带空间坐标的一组接收点观测”。

这类输入更接近集合，而不是规则网格。只要接收点数量可能变化、位置可能不规则、顺序本身不带物理意义，那么把它们硬塞进固定通道的 Conv1d/Conv2d 往往不是最自然的建模方式。

## R 是什么

在表达式

$$
\{(x_i, s_i(t))\}_{i=1}^{R}
$$

中，`R` 表示当前一个样本里的接收点数量。

- `x_i` 是第 `i` 个接收点的位置
- `s_i(t)` 是第 `i` 个接收点测到的一整条时间信号

如果某个样本有 8 个接收点，那么 `R = 8`；另一个样本有 13 个接收点，那么 `R = 13`。`R` 不必固定。

## 为什么 Conv1d/Conv2d 不够自然

### 1. 它们默认输入是固定形状张量

Conv1d 通常要求输入形状稳定，例如：

- 单点输入：`(B, 1, T)`
- 多点通道堆叠：`(B, R, T)`

一旦 `R` 改变，第一层卷积的输入通道数也会变化，模型参数无法直接复用。

### 2. 它们默认局部邻域是规则的

卷积隐含两个很强的前提：

- 相邻索引之间的距离关系是统一的
- 同一个卷积核可以在各位置平移复用

如果接收点位置不等间距，或者不同样本的传感器布局不同，这个假设会变弱。

### 3. 它们不擅长处理“可变数量点集”

卷积网络更像是在处理“规则阵列上的离散信号”，而不是“数量可变、位置可变的一组观测点”。

因此，如果未来目标是：

- 接收点数量可以变化
- 接收点位置不一定规则
- 某些接收点可能缺失
- 不想强制插值到固定网格

那么集合架构会更合适。

## DeepSets 为什么更合适

DeepSets 的核心思想是：

1. 先独立编码每个集合元素
2. 再用对称聚合函数做整体汇总

数学形式常写为：

$$
f\left(\{z_1, \dots, z_R\}\right) = \rho\left(\sum_{i=1}^{R} \phi(z_i)\right)
$$

其中：

- `z_i` 可以是一个接收点的完整信息，例如 `(x_i, s_i(t))`
- `\phi` 是单点编码器
- `\sum` 可以替换成平均、最大值、注意力池化等对称操作
- `\rho` 是集合级输出头

### DeepSets 适合当前任务的原因

#### 1. 天然支持可变数量接收点

DeepSets 不要求输入通道数固定。它处理的是“元素列表”，只要能逐点编码并聚合，就能接受不同大小的集合。

这意味着：

- 一个样本 8 个接收点可以训练
- 另一个样本 12 个接收点也可以训练
- 模型结构本身不需要为不同 `R` 重建

#### 2. 对输入顺序不敏感

集合里的元素没有天然顺序，DeepSets 的对称聚合保证：

$$
f(\{z_1, z_2, z_3\}) = f(\{z_3, z_1, z_2\})
$$

这很符合传感器集合的物理本质，因为数组里的排列顺序不应决定系统物理意义。

#### 3. 不要求规则网格

只要把位置 `x_i` 作为输入特征之一，模型就能学习“传感器位置与信号形态之间的关系”，而不需要先把所有接收点插值到固定空间网格。

#### 4. 更容易容忍缺失传感器

如果某个接收点缺失，DeepSets 只是少一个集合元素，而不是整个输入维度崩掉。

#### 5. 改造成本相对低

如果目标是尽快把当前项目从固定单点输入升级到“多点可变输入”，DeepSets 是一个合理的第一站：

- 架构比 Transformer 简单
- 参数量更可控
- 更容易稳定训练
- 更适合先验证数据组织方式是否可行

### DeepSets 的局限

DeepSets 的短板也很明确：

- 更偏向“单点编码后汇总”
- 对点与点之间的关系建模偏弱
- 对传播时延、相位差、邻近点协同等关系利用不够充分

所以它更适合作为“先把可变点集建模跑通”的方案，而不是最终极限表达能力方案。

## Set Transformer 为什么更合适

Set Transformer 也是处理集合输入，但它比 DeepSets 更进一步：

- DeepSets 强调每个点独立编码后再汇总
- Set Transformer 强调集合内部元素之间的交互

它通常通过自注意力机制，让每个接收点在编码时可以参考其他接收点的信息。

### Set Transformer 适合当前任务的原因

#### 1. 它能显式建模接收点之间的关系

多接收点超声观测不是互相独立的，常见关系包括：

- 到达时间差
- 相位差
- 振幅衰减关系
- 由同一个反射体引起的相关结构

这些关系对波传播建模非常关键，而注意力机制正适合学习这类依赖。

#### 2. 它更适合不规则传感器布局

在不规则布局下，“邻居”不再能简单依赖数组索引定义。Set Transformer 只要拿到每个点的特征和坐标，就可以通过注意力自动学习“谁该关注谁”。

#### 3. 它能自动判断哪些接收点更重要

有些接收点可能更靠近有效回波区域，有些点主要记录背景或噪声。Set Transformer 可以通过注意力权重提升关键接收点的重要性，而不必简单平均所有点。

#### 4. 它更适合复杂传播模式

如果后续目标不只是做去噪，还包括：

- 反射模式分析
- 缺陷定位相关建模
- 多路径传播特征提取

那么 Set Transformer 的表达能力通常比 DeepSets 更强。

### Set Transformer 的代价

它的优势伴随明显代价：

- 模型更复杂
- 训练更慢
- 数据量要求更高
- 调参难度更大
- 如果数据规模不够，可能更容易过拟合

因此，它更适合在数据组织和训练流程已经清楚之后，作为高表达能力方案推进。

## 两种架构的直观区别

### DeepSets 的思路

1. 每个接收点先单独看自己的观测
2. 每个点提取一个局部表示
3. 把所有点的表示做求和或平均
4. 再输出整体结果或逐点结果

可以理解成“先各自发言，再统一汇总”。

### Set Transformer 的思路

1. 每个接收点先得到初始表示
2. 每个点在编码时可参考其他所有点
3. 模型自动学习哪些点之间关系重要
4. 再做集合级推断或逐点重建

可以理解成“大家先互相讨论，再统一得出结论”。

## 对当前项目的实际建议

### 如果优先级是尽快完成迁移

优先考虑 DeepSets。

原因：

- 实现简单
- 对可变 `R` 支持天然
- 便于验证数据结构设计
- 可以先建立一版可靠基线

### 如果优先级是最大化利用多接收点之间的物理联系

优先考虑 Set Transformer。

原因：

- 更擅长建模接收点之间关系
- 对不规则空间分布更友好
- 更适合复杂传播结构和全局依赖

### 更务实的路线

在工程上，比较稳的策略通常是：

1. 先用 DeepSets 跑通数据表示、训练流程和损失设计
2. 再升级到 Set Transformer，比较性能、稳定性和泛化差异

## 与“真正 PINN”的关系

这两种架构并不会自动让模型变成严格 PDE PINN。它们解决的是“输入数据结构更接近点集而不是规则网格”这个问题。

如果未来要做更严格的物理约束，可以在这两种架构上额外加入：

- 基于坐标的物理残差
- 邻点之间的空间差分残差
- 传播时延一致性约束
- 到达时间与波速的几何一致性约束

也就是说，这两种架构更合适，是因为它们更适合承载多接收点、可变点集、不规则布局这一类数据，而不是因为它们天然就是 PINN。

## 一句话总结

- DeepSets 更适合把“可变数量接收点集合”这个问题先稳定建起来。
- Set Transformer 更适合进一步建模接收点之间的传播关系和全局依赖。
- `R` 就是一个样本中的接收点数量，可以变化，不需要固定。
